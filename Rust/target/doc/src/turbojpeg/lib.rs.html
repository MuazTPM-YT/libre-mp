1//! Rust bindings for TurboJPEG, which provides simple and fast operations for JPEG images:
2//!
3//! - [Compression][compress()]: encode images into JPEG.
4//! - [Decompression][decompress()]: decode JPEGs into pixels.
5//! - [Lossless transformations][transform()]: apply basic geometric transformations (rotate, mirror,
6//! ...) without going through decompression and compression, so that the image does not lose
7//! quality.
8//! - [Compression from YUV][compress_yuv()]: encode JPEG from YUV, both in pixel format and in
9//! [planar format][compress_yuv_planes()].
10//! - [Decompression into YUV][decompress_to_yuv()]: decode JPEG into YUV (YCbCr), without
11//! performing the color transform into RGB.
12//! - [Fractional scaling during decompression][Decompressor::set_scaling_factor()]: save time and
13//! memory by downscaling or upscaling the JPEG image while decoding.
14//!
15//! # Integration with image-rs (version 0.24)
16//! 
17//! To easily encode and decode images from the [`image`][image-rs] crate (version 0.24), please
18//! enable the optional dependency `"image"` of this crate in your `Cargo.toml`. Then you can use
19//! the functions [`decompress_image()`][crate::decompress_image] and
20//! [`compress_image()`][crate::compress_image]:
21//! 
22//! ```
23//! # #[cfg(feature = "image")] {
24//! // read JPEG data from file
25//! let jpeg_data = std::fs::read("examples/parrots.jpg")?;
26//!
27//! // decompress `jpeg_data` into an `image::RgbImage`
28//! let image: image::RgbImage = turbojpeg::decompress_image(&jpeg_data)?;
29//!
30//! // compress `image` into JPEG with quality 95 and 2x2 chrominance subsampling
31//! let jpeg_data = turbojpeg::compress_image(&image, 95, turbojpeg::Subsamp::Sub2x2)?;
32//!
33//! # }
34//! # Ok::<(), Box<dyn std::error::Error>>(())
35//! ```
36//! 
37//! This crate supports these specializations of [`image::ImageBuffer`]:
38//! 
39//! - [`image::RgbImage`]
40//! - [`image::RgbaImage`] (JPEG does not support alpha channel, so alpha is ignored when encoding
41//! and set to 255 when decoding)
42//! - [`image::GrayImage`]
43//! 
44//! [image-rs]: https://docs.rs/image/*/image/index.html
45//!
46//! # The [`Image`] type
47//!
48//! For more advanced usage of TurboJPEG, you will need to use the [`Image`] type. This is a simple
49//! struct that contains the geometry of the image (width, height and pitch/stride), pixel format
50//! (such as RGB, ABGR or grayscale) and the pixel data itself.
51//!
52//! [`Image`] is parameterized by the pixel data container, so you will use `Image<&[u8]>` as input
53//! argument for compression, `Image<&out [u8]>` as output argument for decompression, and you may
54//! also find `Image<Vec<u8>>` useful as an owned container of image data in you application.
55//!
56//! # Operations
57//!
58//! - **Decompress** images from JPEG using [`decompress()`] or [`Decompressor`].
59//! - **Compress** images into JPEG using [`compress()`] or [`Compressor`].
60//! - **Transform** images without recompression using [`transform()`] or [`Transformer`]. The
61//! transformations are described in the [`Transform`] struct.
62//! - **Read header** of JPEG image to get its size without decompression using
63//! [`Decompressor::read_header()`] or [`read_header()`].
64//! - **Decompress** images **into YUV** using [`decompress_to_yuv()`] or [`Decompressor`].
65//! - **Compress** images **from YUV** using [`compress_yuv()`] or [`Compressor`].
66//! 
67//! # The [`OutputBuf`] and [`OwnedBuf`] types
68//!
69//! During compression, we need to write the produced JPEG data into some memory buffer. You have
70//! two options:
71//!
72//! - Write the data into a mutable slice (`&mut [u8]`) that you already allocated and initialized.
73//! This has the disadvantage that you must allocate all memory up front, so you need to make the
74//! buffer very large to ensure that it can hold the compressed image even in the worst case, when
75//! the compression does not reduce the image size at all. You will also need to initialize the
76//! memory to comply with the Rust safety requirements.
77//!
78//! - Write the data into a memory buffer managed by TurboJPEG. This has the advantage that
79//! TurboJPEG can automatically resize the buffer, so we don't have to conservatively allocate and
80//! initialize a large chunk of memory, but we can let TurboJPEG grow the buffer as needed. This
81//! kind of buffer is exposed as the [`OwnedBuf`].
82//!
83//! To handle both of these cases, this crate provides the [`OutputBuf`] type, which can hold
84//! either a `&mut [u8]` or an `OwnedBuf`.
85//!
86//! # Features
87//!
88//! - `image`: enables the optional dependency on the [`image`][image-rs] crate.
89//! - `pkg-config`: uses pkg-config to find the `libturbojpeg` library.
90//! - `bindgen`: uses [bindgen] to generate the `libturbojpeg` bindings.
91//!
92//! [bindgen]: https://rust-lang.github.io/rust-bindgen/
